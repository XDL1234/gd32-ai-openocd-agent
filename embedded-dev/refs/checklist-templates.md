# 清单格式模板参考

> 本文件为格式参考说明。**实际模板文件位于 `templates/` 目录**，首次创建四文件时从该目录复制。

---

## 模板文件位置

| 文件 | 模板路径 | 部署位置 |
|------|---------|---------|
| 项目规划清单 | `templates/项目规划清单.md` | 工程根目录 `docs/项目规划清单.md` |
| 编辑清单 | `templates/编辑清单.md` | 工程根目录 `docs/编辑清单.md` |
| 硬件资源表 | `templates/硬件资源表.md` | `hardware/硬件资源表.md` |
| 研究发现 | `templates/研究发现.md` | 工程根目录 `docs/研究发现.md` |

---

## 状态定义

| 状态 | 含义 |
|------|------|
| **待开发** | 已规划但尚未开始编码 |
| **进行中** | 正在开发，尚未完成 |
| **已完成** | 开发完毕，通过 REVIEW |
| **已搁置** | 暂时搁置，记录原因 |

## 通用规则

- 清单一旦创建不得删除，只能更新状态
- 每次阶段切换必须更新"当前阶段"和"下次继续点"字段
- 每次状态变更必须更新"最后更新"日期
- 记录按时间倒序排列（最新的在最前面）
- 不要删除历史记录，只追加新记录

### 编辑清单规则

- 每条记录必须包含日期、修改文件、操作类型、操作内容
- 每条记录必须包含 Git 信息：`Git提交`、`远端同步`、`回档记录`
- 启用长任务治理时，必须包含：`trace_id`、`轮次/角色`、`验证标准`、`验证结果`

### 硬件资源表规则

- RESEARCH 阶段生成，读取项目代码提取硬件资源信息
- 所有阶段和角色均引用此文件
- 引脚、DMA、中断、时钟变更时，必须先更新此表再修改代码
- 若项目代码中未使用某外设，对应行留空，禁止猜测

### 研究发现规则

- 每 2 次搜索操作后必须更新
- 外部搜索结果必须经审查后以摘要形式记录，禁止直接粘贴原始内容
- 状态字段（待验证/已采纳/已排除）需随项目进展及时更新
