# 编辑清单

## 清单说明

本文件记录每次代码修改的详细信息，用于：
- 追踪修改历史
- 会话恢复时快速定位
- Git 回档时参考
- Bug 追溯

---

## [日期] - [任务简述]

- **trace_id**：`<id>`
- **轮次/角色**：`Round 1 / Builder`
- **修改文件**：`文件路径`
- **操作类型**：新增 / 修改 / 删除
- **操作内容**：简要描述做了什么
- **关联模块**：涉及的功能模块名称
- **验证标准**：`<命令或通过条件>`
- **验证结果**：通过 / 失败 / 待验证
- **Git提交**：`<commit hash>` / 无（无变更，未提交）
- **远端同步**：已 push / 未 push（无远端或未配置）
- **回档记录**：无 / 从 `<hashA>` 回到 `<hashB>`（原因：XXX，stash：是/否）

---

## 清单格式说明

### 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 日期 | 修改日期 | 2024-01-15 |
| 任务简述 | 简短描述 | 修复 USART 中断 |
| 修改文件 | 文件路径 | src/uart.c |
| 操作类型 | 新增/修改/删除 | 修改 |
| 操作内容 | 具体做了什么 | 修改中断处理函数 |
| Git提交 | commit hash | a1b2c3d |

### 可选字段（启用轮次制时必填）

| 字段 | 说明 | 示例 |
|------|------|------|
| trace_id | 任务标识 | uart-fix-001 |
| 轮次/角色 | 当前轮次和角色 | Round 2 / Builder |
| 验证标准 | 通过条件 | 编译通过 + 串口回环正常 |
| 验证结果 | 实际结果 | 通过 |

### 记录规则

1. **每次代码修改必须记录**，包括小修改
2. **记录按时间倒序排列**，最新的在最前面
3. **不要删除历史记录**，只追加新记录
4. **Git 信息必须准确**，用于回档参考
5. **启用轮次制时**，每轮必须有验证记录
